# BUG-067: `--diff-filter=ACMR` excludes deletions — delete+docs pushes/commits classify as docs-only and delete-only pushes skip e2e entirely

- **Severity:** Low
- **Category:** correctness
- **Status:** CONFIRMED
- **Files touched:** `.githooks/pre-push`, `.githooks/pre-commit`
- **Roadmap:** none
- **Assigned:** _(left blank for the dev team)_

## Summary
Both hooks classify the change set with `--diff-filter=ACMR`, which omits D (deleted) paths. In pre-push, a push whose commits delete a source file (which breaks every importer at build time) and touch one .md file produces a `changed` list containing only the doc → 'docs-only push — skipping e2e matrix.' A purely delete-only push produces an empty list → 'no new commits to test — skipping e2e.' In pre-commit (line 19), a staged source-file deletion plus a doc edit is likewise classified docs-only, skipping typecheck/lint, so a build-breaking commit passes the cheap gate. Deleting code is a runtime/build-surface change and must not be filtered out of gate classification.

## Where
`.githooks/pre-push:28` — the per-ref `while read` loop's change-set accumulation (`changed="$changed$(git diff --name-only --diff-filter=ACMR "$range" 2>/dev/null)"`); same filter in `.githooks/pre-commit:19`

## How it triggers
Commit/push that deletes any src/ file (e.g. removing a module still imported elsewhere) accompanied only by docs/.md edits — both hooks skip their checks; CI's `pr` job is then the first thing to catch it (or nothing, for e2e-only regressions).

## Verification evidence
Demonstrated in a temp repo: a commit that deletes src2.ts and adds note.md gives `git diff --name-only --diff-filter=ACMR HEAD~1..HEAD` → `[note.md]` only (unfiltered diff shows both `note.md src2.ts`). Fed through .githooks/pre-push:44 (`grep -qvE '(^docs/|\.md$)'` over surviving paths) that push prints 'docs-only push — skipping e2e matrix'; a purely delete-only push yields an empty `changed` → line 38-40 'no new commits to test' skip. Pre-commit (.githooks/pre-commit:19-24) has the same filter, so a staged src-file deletion + a .md edit skips typecheck/lint (only format:check runs). One sub-claim correction: a PURELY delete-only COMMIT does NOT bypass pre-commit — `STAGED` is empty so `[ -n "$STAGED" ]` fails and the hook falls through to the FULL cheap gate (lines 26-29); the bypass needs at least one surviving docs/.md path. The pre-push delete-only and both delete+docs cases hold as claimed. Deleting a still-imported module is a build-breaking change that both local gates misclassify; CI's `pr` check job catches the typecheck breakage later, but e2e-only regressions from deletions are caught by nothing.

## Suggested fix direction
Use `--diff-filter=ACMRD` (or drop the filter) in both hooks' classification — deleted paths must participate in the docs-only decision.
