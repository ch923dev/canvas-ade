# BUG-060: digestPlanning labels a planning board with only text/stroke/image/arrow content as 'Empty board'

- **Severity:** Low
- **Category:** correctness
- **Status:** CONFIRMED
- **Files touched:** `src/renderer/src/lib/digest.ts`, `src/renderer/src/lib/boardSchema.ts`
- **Roadmap:** none
- **Assigned:** _(left blank for the dev team)_

## Summary
digestPlanning counts only `checklist` and `note` elements. The PlanningElement union (boardSchema.ts:105-161) also has `text`, `arrow`, `stroke`, and `image` kinds. A planning board containing e.g. 10 text blocks, drawings, and images produces lines === ['Empty board'] and status 'notes'. The Tier-1 digest feeds the reopen digest panel (T-D2) and the Tier-2 LLM context, so both the user and the model are told a content-rich board is empty — actively wrong information, not just incomplete.

## Where
`src/renderer/src/lib/digest.ts:79` — `digestPlanning` (the `lines.length === 0` → 'Empty board' fallback)

## How it triggers
Open a project whose planning board contains only text/stroke/image/arrow elements (no notes, no checklists); reopen the project and read the digest panel — the board is reported 'Empty board'.

## Verification evidence
Confirmed. digest.ts:70-83 (digestPlanning) filters only `kind === 'checklist'` (:71) and `kind === 'note'` (:72); the fallback `if (lines.length === 0) lines.push('Empty board')` (:79) plus `status = … : 'notes'` (:82) therefore fires for ANY board lacking those two kinds, regardless of other content. boardSchema.ts's PlanningElement union provably includes four more kinds — text (:114), arrow (:126), stroke (:133), image (:153) — so a planning board holding e.g. ten text blocks + drawings + images is reported as lines ['Empty board'] / status 'notes' to both the reopen digest panel (T-D2) and the Tier-2 LLM context: actively wrong information about a content-rich board. The `text` kind shipped later (schema v7, text-font-toolbar) than the digest, consistent with drift rather than intent; digest.test.ts exercises only the truly-empty `elements: []` case. Low: misleading summary, no data loss or broken flow.

## Suggested fix direction
Count the remaining element kinds (text/arrow/stroke/image) into the digest lines — or at minimum gate the 'Empty board' fallback on elements.length === 0.
