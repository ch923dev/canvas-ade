# BUG-040: Saving Settings with a non-local provider unconditionally overwrites (wipes) the stored local baseUrl — the preserve-when-omitted principle applied to maxCallsPerDay (F-B) is not applied to baseUrl

- **Severity:** Low
- **Category:** data-integrity
- **Status:** CONFIRMED
- **Files touched:** `src/main/llmIpc.ts`, `src/renderer/src/canvas/SettingsModal.tsx`, `src/main/llmConfig.ts`
- **Roadmap:** none
- **Assigned:** _(left blank for the dev team)_

## Summary
llm:setConfig deliberately preserves an omitted maxCallsPerDay (`a.maxCallsPerDay ?? existing.maxCallsPerDay`, the F-B fix: 'otherwise every Save silently wipes…') but writes `baseUrl: a.baseUrl` unconditionally. The Settings modal sends `baseUrl: provider === 'local' && baseUrl ? baseUrl : undefined` (SettingsModal.tsx:127), so the supported flow 'configure local + baseUrl → switch to openrouter and Save → later switch back to local' silently erases the stored baseUrl. On returning to local, getProvider() returns null (no baseUrl) → hasProvider:false, the brain silently drops to Tier-1, and the user must rediscover and retype their server URL.

## Where
`src/main/llmIpc.ts:194` — the `llm:setConfig` `ipcMain.handle` callback inside `registerLlmHandlers` (the `cfg` construction at lines 191-196)

## How it triggers
1) Settings: provider=local, baseUrl=http://127.0.0.1:1234/v1, Save. 2) Settings: provider=openrouter, Save (modal sends baseUrl: undefined; config now has baseUrl: undefined). 3) Settings: provider=local — baseUrl field is blank, and saving without retyping it yields a permanently non-functional local provider.

## Verification evidence
The asymmetry is verified in llmIpc.ts:188-196: the F-B comment ('Preserve an already-configured cap when the caller omits it (the Settings modal does): otherwise every Save silently wipes…') applies `?? existing.maxCallsPerDay` to the cap at line 195, but line 194 writes `baseUrl: a.baseUrl` unconditionally. Caller verified at SettingsModal.tsx:124-128: `baseUrl: provider === 'local' && baseUrl ? baseUrl : undefined` — so Save with provider=openrouter sends undefined and the persisted local baseUrl is erased. Round-trip loss verified: on reopening Settings, the modal repopulates baseUrl from llm:status (`if (s.baseUrl) setBaseUrl(s.baseUrl)`, SettingsModal.tsx:80) — now undefined, so the field is blank; and getProvider (llmService.ts:174) returns null for local-without-baseUrl → hasProvider:false → the brain silently drops to Tier-1 until the user rediscovers and retypes the URL. The config file's baseUrl slot is documented local-only (llmConfig.ts:32-33), so preserving it across non-local saves is loss-free, matching the F-B intent. Low: recoverable by retyping one URL, and the blank field is visible when the user returns to the local provider in Settings.

## Suggested fix direction
Apply the same preserve-when-omitted rule: `baseUrl: a.baseUrl ?? existing.baseUrl`.
