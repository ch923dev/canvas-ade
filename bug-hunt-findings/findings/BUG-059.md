# BUG-059: URL draft is clobbered mid-edit: render-time re-sync on board.url change resets draftUrl even while the input is focused

- **Severity:** Low
- **Category:** correctness
- **Status:** CONFIRMED
- **Files touched:** `src/renderer/src/canvas/boards/BrowserBoard.tsx`, `src/renderer/src/canvas/boards/useBrowserAutoConnect.ts`
- **Roadmap:** none
- **Assigned:** _(left blank for the dev team)_

## Summary
The 'adjust state on prop change' block (`if (board.url !== lastUrl) { setLastUrl(...); setDraftUrl(board.url) }`) runs unconditionally, including while the user is actively typing in the URL input. Any external board.url change — the auto-connect detect push (useBrowserAutoConnect runs exactly when the board has no valid http URL, i.e. precisely when the user is likely typing one), a terminal push-to-preview, an MCP configureBoard, or an undo — replaces the user's partially-typed URL mid-keystroke. The user's input is silently lost with no way to recover it.

## Where
`src/renderer/src/canvas/boards/BrowserBoard.tsx:137` — `BrowserBoard` (the render-time "adjust state on prop change" draftUrl re-sync block)

## How it triggers
Create/link a Browser board with an empty or invalid URL and a linked source terminal whose dev server is starting; begin typing a URL into the bb-url-input; within the 1s tick + backoff window detectPorts resolves and updateBoard sets board.url → draftUrl resets under the caret.

## Verification evidence
Confirmed. BrowserBoard.tsx:137-140: `if (board.url !== lastUrl) { setLastUrl(board.url); setDraftUrl(board.url) }` runs unconditionally during every render — there is no focus/isEditing gate anywhere in the component (the input at :253-269 is a plain controlled input bound to draftUrl, committed only on Enter/blur). The colliding writer is real and fires precisely in the typing state: useBrowserAutoConnect.ts:69-73 plans 'detect' when `hasUrl: isHttpUrl(board.url)` is FALSE and previewSourceId is set — i.e. exactly the no-valid-URL state in which a user would be typing a URL — and :104 then sets board.url via updateBoard while the user's keystrokes only mutate draftUrl (board.url unchanged until blur). The next render hits :137 (board.url changed under the unchanged lastUrl) and resets draftUrl under the caret, silently destroying the partial input. Other external writers (terminal push-to-preview, MCP configureBoard, undo) hit the same path. Low: realistic edge, recoverable by retyping.

## Suggested fix direction
Gate the render-time re-sync on the URL input not being focused (track focus state; sync the skipped value on blur), the standard controlled-input-with-external-writer pattern.
