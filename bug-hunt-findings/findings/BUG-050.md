# BUG-050: Note hit/bbox geometry uses the never-updated persisted h:96 while NoteCard auto-sizes — phantom erase band below short notes, unreachable bottoms of tall notes

- **Severity:** Low
- **Category:** correctness
- **Status:** CONFIRMED
- **Files touched:** `src/renderer/src/canvas/boards/planning/elements.ts`, `src/renderer/src/canvas/boards/planning/NoteCard.tsx`, `src/renderer/src/canvas/boards/planning/erase.ts`, `src/renderer/src/canvas/boards/PlanningBoard.tsx`
- **Roadmap:** none
- **Assigned:** _(left blank for the dev team)_

## Summary
makeNote persists h:96 and nothing ever updates it, but NoteCard sets no height — its rendered height is grip padding (18px) + the auto-grown textarea (16px/line). A fresh one-line note renders ~34px tall; a 12-line note renders ~210px. elementBBox's note case returns el.h unconditionally (it ignores the `measured` parameter, unlike text/checklist), and NoteCard is the one card PlanningBoard does not wire onMeasure into (PlanningBoard.tsx:666-680 passes onMeasure to FreeText/ChecklistCard only). eraseHitTest's note case likewise uses el.h (erase.ts:95). So: an erase swipe up to ~70px BELOW a short note's visible bottom edge (96-34+8 tol) deletes it — destructive removal of an element the cursor never touched; the bottom half of a tall note doesn't respond to the eraser or right-click targeting; marquee selection, snapping guides, align/distribute, and the SVG export all operate on the wrong box for any note whose text isn't ~4 lines.

## Where
`src/renderer/src/canvas/boards/planning/elements.ts:295` — `elementBBox` (the `case 'note'` branch, which ignores the `measured` param)

## How it triggers
Create a note, type one short line, click elsewhere; switch to the eraser and swipe through the visually empty area ~40px below the note — the note is deleted. Or fill a note with 12 lines and try to erase / right-click its lower half — no hit.

## Verification evidence
Confirmed. makeNote persists h:96 (elements.ts:25, 58-59) and nothing ever writes note h afterward. NoteCard.tsx styles only width (77-92 — no height/minHeight), with the grip's 9px vertical padding (line 153) plus a freely auto-growing textarea (autosize effect 51-56, lineHeight 16px line 188) — so a one-line note renders ~36px and a 12-line note ~210px. eraseHitTest's note case uses el.h (erase.ts:94-95) and elementBBox's note branch ignores the `measured` param entirely (elements.ts:295-296), unlike the checklist branch (298-303) which got both a measured path and a nominal formula. PlanningBoard.tsx wires onMeasure only into FreeText (694) and ChecklistCard (717) — NoteCard (669-679) gets none, and NoteCardProps has no onMeasure at all. Result: an erase swipe up to ~60px+tol below a short note's visible edge deletes it, the lower half of a tall note ignores eraser/right-click, and marquee/snap/align/export use the wrong box.

## Suggested fix direction
Wire onMeasure into NoteCard and make elementBBox/eraseHitTest's note case consume the measured height (checklist pattern).
