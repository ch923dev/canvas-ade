# BUG-053: Double-click inside the TerminalConfig popover (or preview note) flips the board to recap, leaving the open config pointer-dead

- **Severity:** Low
- **Category:** correctness
- **Status:** CONFIRMED
- **Files touched:** `src/renderer/src/canvas/boards/TerminalBoard.tsx`, `src/renderer/src/canvas/boards/TerminalConfig.tsx`
- **Roadmap:** none
- **Assigned:** _(left blank for the dev team)_

## Summary
The flip stage's onDoubleClick guard only skips targets matching closest('button, input, .ca-port-picker, [data-no-flip]'). The TerminalConfig popover root (className 'nowheel', no data-no-flip) and its &lt;select&gt;, &lt;label&gt; text, and the font-value &lt;span&gt; match none of these, and TerminalConfig only stops onPointerDown/onKeyDown propagation — dblclick bubbles to the stage handler. A double-click on the Shell &lt;select&gt;, a label, or popover padding flips the board to the recap face; the front-face container then gets pointerEvents:'none' (line 967), so the still-open config popover becomes unclickable until the user double-clicks the recap to flip back. The transient '.ca-preview-note' div has the same hole.

## Where
`src/renderer/src/canvas/boards/TerminalBoard.tsx:953` — `TerminalBoard` (the flip stage `<div>`'s `onDoubleClick` exclusion guard)

## How it triggers
Open a terminal board's Configure popover and double-click the Shell select (or any label / padding inside the popover). The board folds to the recap; the config popover is now inert.

## Verification evidence
Confirmed DOM path. The flip stage's onDoubleClick guard (TerminalBoard.tsx:949-958) skips only targets matching closest('button, input, .ca-port-picker, [data-no-flip]'). TerminalConfig's root (TerminalConfig.tsx:78-89) has className 'nowheel' with no data-no-flip and stops only onPointerDown/onKeyDown/onWheel — dblclick bubbles. Its &lt;select&gt; (104-120), the &lt;label&gt; text wrappers (90, 102, 122, 134), the font-value &lt;span&gt; (157), and the popover padding all match none of the four selectors (closest finds no button/input ancestor for them), so a double-click there reaches the stage handler and runs flip.toggle(). When flipped, the front-face container gets pointerEvents:'none' (TerminalBoard.tsx:960-968) and TerminalConfig renders inside it (970-977) → the still-open popover is unclickable until the user double-clicks the recap to flip back. The '.ca-preview-note' div (991-1002) stops only onMouseDown and is likewise outside the exclusion list. The handler's own comment ('a double-click on a control never also flips the board') shows the exclusion list is meant to cover exactly this chrome and misses select/label/span/container.

## Suggested fix direction
Add 'select, label' and a data-no-flip attribute on the TerminalConfig root + .ca-preview-note to the closest() exclusion list.
