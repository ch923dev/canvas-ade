# BUG-012: duplicate → undo → redo of a terminal clone loses its idle-on-mount flag (BUG-033 sweep has no redo counterpart) — the redone clone auto-spawns shell + launchCommand

- **Severity:** Medium
- **Category:** correctness
- **Status:** CONFIRMED
- **Files touched:** `src/renderer/src/store/canvasStore.ts`, `src/renderer/src/canvas/boards/TerminalBoard.tsx`
- **Roadmap:** none
- **Assigned:** _(left blank for the dev team)_

## Summary
`duplicateBoard` adds a terminal clone's id to `idleOnMountIds` so the clone mounts IDLE (locked M-1 rule: cloning must not silently spin up a second agent). `undo()` sweeps the ids of boards that vanish from the present out of the set (the BUG-033 leak fix), but `redo()` has no symmetric re-add — so duplicate → undo → redo restores the clone WITHOUT its idle flag. On remount, `adoptTerminal` finds no parked PTY, `isIdleOnMount(id)` is false, and TerminalBoard takes the auto-spawn branch, spawning a shell and writing the board's `launchCommand` (e.g. `claude`) as the first PTY line — silently starting a second live agent.

## Where
`src/renderer/src/store/canvasStore.ts:702` — `undo()` (the BUG-033 sweep loop at lines 702-705 calling `clearIdleOnMount`; `redo()` at lines 716-734 has no re-add; the flag is set in `duplicateBoard` at line 519)

## How it triggers
Duplicate a terminal board, press Ctrl+Z, press Ctrl+Y (redo). The redone clone auto-spawns its shell + launchCommand instead of mounting idle with the Start affordance.

## Verification evidence
Empirically reproduced against the real store (same bundled-module repro): after duplicateBoard, isIdleOnMount(clone)=true (set at canvasStore.ts:519); after undo() the clone is gone AND the flag is swept — undo's BUG-033 loop (lines 702-705) calls clearIdleOnMount for every board absent from the restored present; after redo() the clone is back in boards but isIdleOnMount(clone)=false — redo() (lines 716-734) has no symmetric re-add. Repro output: 'C5 after redo: clone in boards? true isIdleOnMount= false'. Consumption path is real: TerminalBoard.tsx:495-505 — adoptTerminal resolves adopted:false (the idle clone never spawned a PTY to park), the `isIdleOnMount(board.id)` branch (line 499) is now false, so spawnAllowed=true + launch() (lines 502-503) → window.api.spawnTerminal with the cloned board's launchCommand (line 465), silently starting a second agent — exactly what the M-1 comment above duplicateBoard (lines 517-518: 'cloning must not silently spin up a second agent') forbids. Existing tests (canvasStore.test.ts:184, 196) cover only duplicate+undo, never the redo leg.

## Suggested fix direction
Make redo() re-add idle flags for boards that REAPPEAR (or have undo park the swept ids keyed by snapshot so redo restores them); alternatively sweep lazily on a timer/save instead of inside undo.
