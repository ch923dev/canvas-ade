# BUG-064: MAIN-pushed recap:learned patch goes through updateBoard and silently clears an armed redo stack (and is reverted by any later undo)

- **Severity:** Low
- **Category:** data-integrity
- **Status:** CONFIRMED
- **Files touched:** `src/renderer/src/App.tsx`, `src/renderer/src/store/canvasStore.ts`
- **Roadmap:** none
- **Assigned:** _(left blank for the dev team)_

## Summary
The recap onLearned handler patches agentSessionId/agentTranscriptPath via s.updateBoard(). updateBoard (canvasStore.ts:537-567) treats any diff as a live edit and returns `{ boards, future: [] }` when future is non-empty. The learned patch arrives asynchronously from MAIN (transcript-map watcher), not from a user gesture - so if the user has just undone something (redo armed), a Claude session starting in any terminal board wipes their redo branch at a random moment with zero user action. Secondarily, any later undo wholesale-restores a past snapshot that predates the patch, silently reverting the learned session id/transcript path (breaking the recap until the next learn event).

## Where
`src/renderer/src/App.tsx:55` — `App` component, the Terminal-recap T15 `useEffect` (`onLearned` subscription calling `s.updateBoard(p.boardId, …)`)

## How it triggers
User performs canvas edits, presses Ctrl+Z (future non-empty), and a terminal board's Claude session starts (or the transcript map updates) -> MAIN pushes recap:learned -> redo (Ctrl+Shift+Z) is now a no-op.

## Verification evidence
Code-path proof: App.tsx:51-58 — the recap onLearned handler calls s.updateBoard(p.boardId, { agentSessionId, agentTranscriptPath }) for every MAIN push. Both keys are in PATCHABLE_KEYS.terminal (canvasStore.ts:412-421), so they survive the filter. updateBoard (canvasStore.ts:537-566): the first learn always diffs (b.agentSessionId is undefined vs the pushed string, line 557), so changed=true and line 566 returns `s.future.length ? { boards, future: [] } : { boards }` — the armed redo branch is wiped. The push originates from MAIN's transcript-map watcher (preload index.ts:281-290, 'recap:learned'), not a user gesture, so the trigger is exactly as claimed: user edits, presses Ctrl+Z (future non-empty), a Claude session starts in any terminal → redo silently becomes a no-op. A new session later re-triggers it (new sessionId → diff again); only identical re-pushes are absorbed by the STATE-2 value compare. Secondary effect also real: updateBoard does NOT recordPast, but any later undo restores a whole pre-patch boards snapshot (history applyUndo path), silently dropping the learned sessionId/transcriptPath until the next learn event. Severity Low is right: occasional loss of the redo branch / recap metadata, no data-file corruption.

## Suggested fix direction
Route MAIN-originated metadata patches through a non-history setter (skip the future-clear, e.g. a dedicated patchBoardMeta that also rewrites past/future snapshots or is excluded from snapshots).
