# BUG-029: Untrusted preview content can fire unbounded, gesture-free shell.openExternal calls via window.open (OS browser tab flood / phishing launcher)

- **Severity:** Low
- **Category:** security
- **Status:** CONFIRMED
- **Files touched:** `src/main/preview.ts`
- **Roadmap:** none
- **Assigned:** _(left blank for the dev team)_

## Summary
The per-view `setWindowOpenHandler` calls `openExternalSafe(url)` for every `window.open()` from the previewed page, gated only on scheme (http/https/mailto). Electron does not enforce Chromium's user-activation requirement for window.open, and there is no rate limit, dedupe, or gesture check — so a malicious or compromised page in the preview (explicitly "UNTRUSTED localhost content" per this file's own threat-model comments) can loop `window.open('https://phish.example/fake-login')` and spawn an unbounded stream of OS-browser tabs with zero user interaction. That is both a denial-of-service on the user's desktop and a phishing primitive (the opened page appears in the user's real browser with fully trusted chrome).

## Where
`src/main/preview.ts:268` — `ensure()` (the per-view `wc.setWindowOpenHandler` wiring)

## How it triggers
Any previewed page (or anything it loads — the nav guards allow arbitrary http(s) subframes/redirects) runs `setInterval(() => window.open('https://attacker.example'), 50)` — no user gesture needed; the OS browser is flooded.

## Verification evidence
Code path is direct and unguarded: every window.open from a previewed page hits the per-view setWindowOpenHandler (src/main/preview.ts:268-271) → openExternalSafe (lines 112-116) → shell.openExternal, gated ONLY on scheme (http/https/mailto, isAllowedExternal lines 97-105). There is no rate limit, dedupe, or user-gesture requirement, and Electron does not ship Chromium's popup blocker — window.open without user activation still invokes setWindowOpenHandler. The file's own threat-model comments (lines 92-95, 264-267) establish preview content as UNTRUSTED and window.open as the anticipated attack channel; Bug #23 gated WHAT can be opened but nothing gates HOW OFTEN or WHETHER a gesture occurred. A hostile/compromised localhost page running setInterval(() => window.open('https://attacker.example'), 50) floods the OS browser with real-chrome tabs (phishing-assist + desktop DoS). Low is correct: requires hostile content on the previewed port; annoyance/phishing-assist, not code execution.

## Suggested fix direction
Require a recent user gesture (or rate-limit/debounce per view) before forwarding window.open targets to shell.openExternal.
