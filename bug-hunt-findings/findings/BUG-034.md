# BUG-034: readTranscriptTail discards readSync's bytesRead over Buffer.allocUnsafe — a shrink-race short read decodes uninitialized MAIN-heap bytes into the milestone-extraction input

- **Severity:** Low
- **Category:** correctness
- **Status:** CONFIRMED
- **Files touched:** `src/main/agentTranscript.ts`, `src/main/summaryLoop.ts`
- **Roadmap:** none
- **Assigned:** _(left blank for the dev team)_

## Summary
readTranscriptTail allocates `Buffer.allocUnsafe(len)` (len from a prior fstatSync) and calls readSync without using its returned byte count. If the file shrinks between fstat and read (TOCTOU on a file an external process owns), or the read returns short, the tail of the buffer keeps whatever was previously in MAIN's heap, and `buf.toString('utf8')` includes it in the string handed to extractMilestones — whose output feeds buildRecapInput and egresses to the LLM. MAIN's heap holds decrypted API keys (llmKeyStore via safeStorage) and transcript content, so uninitialized-memory inclusion in an egress-bound string is the wrong failure mode even if random heap bytes rarely parse as valid JSONL milestone lines.

## Where
`src/main/agentTranscript.ts:72` — `readTranscriptTail()` (the `readSync(fd, buf, 0, len, start)` whose return value is discarded)

## How it triggers
Transcript file truncated/replaced by an external process between fstatSync and readSync inside the same call (claude normally appends, so rare — e.g. user deletes/rotates ~/.claude transcripts while a recap fires).

## Verification evidence
Verified at src/main/agentTranscript.ts:71-73: `const buf = Buffer.allocUnsafe(len); readSync(fd, buf, 0, len, start); return buf.toString('utf8')` — the readSync return value (actual bytes read) is discarded and the FULL buffer is decoded. `len` comes from fstatSync at line 67; if an external process truncates the transcript between fstat and read (the fd stays valid; ftruncate by another process shortens what readSync can return), the tail of the allocUnsafe buffer is uninitialized prior heap content and is included in the returned string. Caller path confirmed: src/main/index.ts:322 `extractMilestones(readTranscriptTail(path), ...)` → milestones → summaryLoop.ts:508 buildRecapInput → LLM egress. Mitigation confirmed too: extractMilestones (agentTranscript.ts:39-43) JSON.parses per line and drops malformed lines, so random heap bytes almost never survive to egress — but heap reuse can contain previously-read transcript/JSONL content that IS valid, and decoding uninitialized memory into an egress-bound string is the defect regardless. No test covers the shrink race (agentTranscript.test.ts only tests stable files). Severity stays Low: narrow TOCTOU window on a file claude only appends to, plus the JSON-parse filter.

## Suggested fix direction
Use the readSync return value: `const n = readSync(...); return buf.toString('utf8', 0, n)` (or Buffer.alloc).
