# BUG-003: Packaged-build recap hook is doubly broken: process.execPath without ELECTRON_RUN_AS_NODE launches a full second app instance, and the baked script path points inside app.asar

- **Severity:** High
- **Category:** correctness
- **Status:** CONFIRMED
- **Files touched:** `src/main/index.ts`, `src/main/agentRecapMap.ts`, `src/main/hooks/recordSession.js`, `electron-builder.yml`
- **Roadmap:** partially covered by Phase 5 — Packaging & release (see cross-ref)
- **Assigned:** _(left blank for the dev team)_

## Summary
`installRecapHook` bakes `nodePath: process.execPath` and a `__dirname`-relative `recordSession.js` path into the project's `.claude/settings.local.json`. In a packaged build (a) `process.execPath` is the app exe with no default_app, so each claude SessionStart in a consented project ignores the positional .js argument and boots a complete second Canvas ADE instance (no `requestSingleInstanceLock` anywhere), the hook blocks until claude's timeout, and the session map is never recorded — recap silently dead. (b) Independently, the baked script path resolves inside `app.asar`; electron-builder asarUnpacks the file but index.ts never applies the standard `app.asar → app.asar.unpacked` rewrite, so even a real node could not read it. Dev builds mask both, which is why the 2026-06-08 live verification passed.

## Where
`src/main/index.ts:354` — `nodePath: process.execPath` baked into the `installRecapHook` call inside the project-open re-install callback passed to `registerProjectHandlers` (same bake repeated at index.ts:381 in the `registerRecapHandlers` onDecision callback; script path computed at index.ts:201)

## How it triggers
Any packaged build (electron-builder nsis/dmg/AppImage): enable recap consent on a project, then start a claude session in it. Dev builds mask both bugs (the bare electron binary's default_app runs the script, and out/main/hooks exists on disk), which is why the 2026-06-08 live verification passed.

## Verification evidence
Confirmed, both halves — packaged builds only (dev verified-green is fully explained). (a) index.ts:354 and :381 bake `nodePath: process.execPath` into the hook; agentRecapMap.installRecapHook (:65-68) writes `{type:'command', command: opts.nodePath, args:[scriptPath, mapPath]}` verbatim with NO env — grep over src/ finds ZERO occurrences of ELECTRON_RUN_AS_NODE. In a packaged build process.execPath is the app exe; a packaged Electron binary has no default_app, so the positional .js arg is ignored and the full Canvas ADE app boots instead. grep finds ZERO `requestSingleInstanceLock` in src/, so every claude SessionStart in a consented project launches a complete second app instance (window, MCP server, local server), the hook blocks until claude's hook timeout, and recordSession.js never runs → the session map is never written → recap silently dead. (b) Independently, index.ts:201 computes `recordScript = join(__dirname,'hooks','recordSession.js')`; packaged __dirname is inside resources/app.asar/out/main. electron-builder.yml:20-23 asarUnpacks '**/hooks/recordSession.js' to app.asar.unpacked, but grep finds NO '.replace("app.asar", "app.asar.unpacked")' (or any asar-path rewrite) anywhere in src/ — the baked string still points through the app.asar archive file, which an external process resolves as a file-not-directory (ENOTDIR); the yml comment's claim that 'the recordScript path computed in main resolves to a real file' is wrong for the baked external path. Dev masks both.

## Suggested fix direction
Bake env ELECTRON_RUN_AS_NODE=1 into the hook command (or detect/ship a real node) AND rewrite the script path with the standard app.asar → app.asar.unpacked idiom when app.isPackaged.
