# BUG-003: Clicking a board resize handle without dragging leaves nodeGesture stuck true → live Browser previews freeze to snapshots and the next board-drag re-occludes them

- **Severity:** Medium
- **Category:** concurrency
- **Status:** CONFIRMED
- **Files touched:** `src/renderer/src/canvas/BoardNode.tsx`
- **Roadmap:** none
- **Assigned:** _(left blank for the dev team)_

## Summary
BoardNode wires NodeResizer.onResizeStart→setNodeGesture(true) and onResizeEnd→setNodeGesture(false), but React Flow's XYResizer fires onResizeStart on every pointer-down of a handle while gating onResizeEnd behind resizeDetected (only set true on real movement). A pure click on a visible handle therefore sets nodeGesture=true with no matching false, so BrowserPreviewLayer detaches every live WebContentsView to a static HTML snapshot and never reattaches via that path. Because setNodeGesture dedups equal values, the stuck flag also no-ops the NEXT real node-drag's setNodeGesture(true), so beginMotion is skipped and the dragged board occludes the live preview — re-opening the exact occlusion bug nodeGesture exists to prevent.

## Where
- src/renderer/src/canvas/BoardNode.tsx:93-97 (onResizeStart/onResizeEnd on NodeResizer); src/renderer/src/store/previewStore.ts:77 (setNodeGesture dedup); src/renderer/src/canvas/boards/BrowserPreviewLayer.tsx:435-442 (nodeGesture effect → beginMotion/endMotion); node_modules/@xyflow/system/dist/esm/index.js:3407 + 3496-3499 (XYResizer start fires unconditionally, end gated on resizeDetected)

## How it triggers
With at least one live Browser board on canvas, hover or select any board so its resize handles appear, then press-and-release (click, no drag past the dimension-change threshold) any resize handle. onResizeStart fires → setNodeGesture(true); onResizeEnd never fires because resizeDetected stayed false → nodeGesture remains true. BrowserPreviewLayer's effect runs beginMotion(), capturing+detaching every live preview to a snapshot; no endMotion ever arrives on this path, so all previews stay frozen as static images and the gesture flag stays armed. A subsequent board drag then fails to detach (dedup no-op) and paints its native view over the live preview. Recovery happens only incidentally via a later real resize (onResizeEnd) or any camera pan/zoom (useOnViewportChange.onEnd→endMotion reattaches) — but a camera move does NOT clear nodeGesture, leaving the drag-occlusion regression armed.

## Suggested fix direction
Make nodeGesture self-healing rather than relying on the asymmetric onResizeStart/onResizeEnd pair. Options: (a) only flip nodeGesture to true on the first onResize (real movement) instead of onResizeStart, so a pure click never arms it; (b) guard against a stuck flag by clearing nodeGesture on pointer-up of the handle (attach a one-shot pointerup/window mouseup listener when onResizeStart arms it); or (c) have camera onMoveEnd (endMotion) also reset usePreviewStore nodeGesture to false so any incidental recovery also disarms the compounding drag-occlusion regression. Option (a) is the cleanest and mirrors XYResizer's own resizeDetected semantics.
