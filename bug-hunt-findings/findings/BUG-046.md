# BUG-046: startPlacement has no pointer-button filter — right/middle-click while a placement tool is armed creates a phantom board (native-menu half of the claim refuted: Electron shows no default context menu)

- **Severity:** Low
- **Category:** correctness
- **Status:** CONFIRMED
- **Files touched:** `src/renderer/src/canvas/hooks/useBoardPlacement.ts`, `src/renderer/src/canvas/Canvas.tsx`
- **Roadmap:** none
- **Assigned:** _(left blank for the dev team)_

## Summary
startPlacement (the capture overlay's onPointerDown) checks only `tool === 'select'` and never inspects e.button or e.isPrimary. A right-button pointerdown on the placement-capture overlay starts the ghost drag, and the right-button pointerup fires the window 'pointerup' handler, which commits addBoard (default-size on a short press, or an exact rect if dragged). The overlay also doesn't suppress contextmenu, so on Windows the native context menu pops at the same time the phantom board is created and the tool disarms. Same for middle-click (button=1).

## Where
`src/renderer/src/canvas/hooks/useBoardPlacement.ts:70` — `startPlacement` in `useBoardPlacement` (overlay wired in `CanvasInner`, Canvas.tsx:865-878)

## How it triggers
Arm any board type from the dock, then right-click (or middle-click) anywhere on the canvas.

## Verification evidence
Code-path proof. useBoardPlacement.ts:70-114 — startPlacement checks only `tool === 'select'` (line 72); no `e.button`/`e.isPrimary` guard anywhere in the hook. The overlay (Canvas.tsx:865-878) attaches it bare: `<div className="placement-capture" onPointerDown={startPlacement}>` with no onContextMenu/preventDefault. A right-button press is a zero→one buttons transition, so pointerdown fires (React forwards all buttons), arming the window pointerup at lines 106-107; the right-button release fires window pointerup → onUp (lines 85-105) unconditionally commits `addBoard` (default-size via isClickGesture, line 88-91). useBoardPlacement.test.tsx has zero button-related coverage (grep for `button|isPrimary` = no matches). REFUTED sub-claim: 'the native context menu pops' — grep of src/main for context-menu/contextmenu finds nothing, and Electron renders no default context menu, so only the phantom board + tool disarm happen.

## Suggested fix direction
Early-return in startPlacement unless `e.button === 0 && e.isPrimary` (optionally treat right-click as cancel, matching Esc).
