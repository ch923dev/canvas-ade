# BUG-072: tsconfig.web.json grants project-wide Node ambient types to the sandboxed renderer, blinding typecheck to 'renderer never touches Node' violations

- **Severity:** Low
- **Category:** other
- **Status:** CONFIRMED
- **Files touched:** `tsconfig.web.json`
- **Roadmap:** none
- **Assigned:** _(left blank for the dev team)_

## Summary
tsconfig.web.json compiles the sandboxed renderer (`src/renderer/src/**`) with `"types": ["node"]`, so `process`, `Buffer`, `__dirname`, `require`, NodeJS.* globals and `import fs from 'fs'`-style imports all typecheck cleanly in renderer code — even though the renderer runs with `sandbox: true` / `nodeIntegration: false` and any such usage is a runtime failure (or, if it ever 'works', an invariant breach). Since `pnpm typecheck` is the project's primary static gate (pre-commit + CI `check` job), the one tool positioned to catch an accidental Node-API leak into the renderer is blinded. If specific files need NodeJS.Timeout or similar, scope it (e.g. `ReturnType<typeof setTimeout>`, or per-file triple-slash references) instead of an ambient project-wide grant.

## Where
`tsconfig.web.json:21` — `compilerOptions.types` of the renderer (web) TypeScript project

## How it triggers
Add `const v = process.env.FOO` or `import { readFileSync } from 'fs'` to any renderer component — typecheck, lint, and the pre-commit gate all pass; the defect only surfaces at runtime in the sandboxed window (undefined global / unresolvable module).

## Verification evidence
tsconfig.web.json:21 `"types": ["node"]` with include = renderer src + vendor + preload .d.ts only (lines 23-28). Faulty path demonstrated with the repo's own tsc (node_modules/typescript) on a temp probe file under the exact web compiler options (strict, ES2022, Bundler resolution, DOM libs): with `--types node`, `import { readFileSync } from 'fs'`, `process.env.FOO`, `Buffer.from('x')`, and `__dirname` ALL typecheck clean (output: 'TYPECHECK PASSED'); with `types: []` the identical file produces 4 errors (TS2307 'Cannot find module fs', TS2591 'Cannot find name process/Buffer', TS2304 '__dirname'). Refutation attempt failed: grep across src/renderer and src/vendor finds ZERO uses of `NodeJS.`, `process.`, `Buffer`, `__dirname`, or `require(` — nothing in the renderer project legitimately needs the grant, so it is purely a blinding ambient. Since `pnpm typecheck` (pre-commit + CI check job) is the primary static gate and the renderer runs sandbox:true/nodeIntegration:false, an accidental Node-API leak passes every automated gate and only fails at runtime in the sandboxed window. Not a runtime bug today (no current violations exist) — an enforcement gap for a documented security invariant, Low.

## Suggested fix direction
Drop `types: ["node"]` from tsconfig.web.json (set `types: []`); if a future file needs timer types use `ReturnType<typeof setTimeout>` rather than re-granting Node ambients project-wide.
