# BUG-017: Board stays stuck in hovered=true after zooming across the LOD boundary with the cursor over it

- **Severity:** Low
- **Category:** correctness
- **Status:** CONFIRMED
- **Files touched:** `src/renderer/src/canvas/BoardNode.tsx`
- **Roadmap:** none
- **Assigned:** _(left blank for the dev team)_

## Summary
BoardNode keeps hover in local useState toggled by onMouseEnter/onMouseLeave on an inner div that only exists in the full-chrome render. For browser/planning boards the LOD branch early-returns a pointerEvents:'none' card that never mounts that div, so when the camera crosses below LOD_ZOOM (0.4) with the cursor over the board, the unmount fires no mouseLeave and hovered stays true. On zoom-in the board renders a spurious hover border and visible resize handles (NodeResizer isVisible={selected||hovered}) with the pointer elsewhere. It is a transient, recoverable visual glitch — moving the cursor onto and off the board clears it.

## Where
- src/renderer/src/canvas/BoardNode.tsx:59 (hovered useState); :67-80 (LOD early-return, pointerEvents:'none', no hover div); :90 (NodeResizer isVisible={selected||hovered}); :100-104 (onMouseEnter/onMouseLeave div, full-chrome only). Consumed at src/renderer/src/canvas/BoardFrame.tsx:208 (hovered drives border color).

## How it triggers
With an unselected browser or planning board on screen at zoom >= 0.4: move the cursor over the board so onMouseEnter sets hovered=true (border + resize handles appear). While the cursor stays over the board, zoom the camera out past 40% (e.g. Ctrl/Cmd+wheel zoom-to-cursor) so isLod(transform[2]) flips true and BoardNode takes the LOD early-return. The hover div unmounts under the cursor, which fires no mouseLeave, so hovered remains true; the LOD card sets pointerEvents:'none' so no further pointer events arrive. Zoom back in above 40%: the full-chrome branch renders with stale hovered=true, showing the hover border (BoardFrame.tsx:208) and resize handles (BoardNode.tsx:90) even with the pointer off the board. Recovers once the cursor enters and then leaves the board.

## Verification evidence
Code-path trace against current code (post-48c6500): (1) Node instances are keyed solely by stable b.id (Canvas.tsx:91-101 nodes useMemo, no key change on zoom), so the same BoardNode fiber survives the LOD transition and its useState(hovered) (BoardNode.tsx:59) persists. (2) The lod boolean is derived via useStore((s)=>isLod(s.transform[2])) (BoardNode.tsx:58); isLod returns zoom<0.4 (canvasView.ts:24-26 LOD_ZOOM=0.4). (3) For board.type !== 'terminal' the lod branch returns at BoardNode.tsx:67-80 — a div with pointerEvents:'none' (line 69) wrapping BoardFrame lod, rendered BEFORE and INSTEAD of the onMouseEnter/onMouseLeave div at lines 100-104, so that div is unmounted at LOD. (4) React does not synthesize onMouseLeave/onMouseOut when an element unmounts under a stationary cursor (its synthetic mouse-leave events derive from real DOM mousemove/mouseout; removing the node dispatches no leave) — a long-standing, stable React 18 behavior — so setHovered(false) (line 102) is never called on LOD entry. (5) BoardNode has NO useEffect and the LOD card is pointerEvents:'none', so nothing resets hovered while at LOD. (6) On return, hovered=true flows to BoardFrame.tsx:208 (hovered ? 'var(--border)') for the border and to NodeResizer isVisible={selected||hovered} (BoardNode.tsx:90), so an unselected board shows handles + hover border with the cursor off it. (7) Searched docs/bug-hunt-findings.md (no entry for mouseleave/stale-hover/LOD hover) — not among the 50 remediated bugs. No jsdom/RTL component tests exist (vitest covers only pure-logic modules), so nothing exercises or contradicts this path. Terminal boards are exempt because they fall through to the full-chrome return that keeps the hover div mounted at LOD.

## Suggested fix direction
Clear hover on LOD entry/exit instead of relying on a mouseLeave that can't fire: add a useEffect in BoardNode that calls setHovered(false) when lod becomes true (or gate the hover-driven UI on !lod && hovered). Equivalently, derive the resizer/border from a hover signal that resets when the hover div unmounts, or move the onMouseEnter/onMouseLeave to a wrapper that stays mounted across the LOD transition so leave can fire normally.
