# BUG-061: OVERVIEW_FRAME omits maxZoom, so it inherits the flow's Z_MAX 2.5 and frames TIGHTER than Fit (maxZoom 2) for small clusters

- **Severity:** Low
- **Category:** correctness
- **Status:** CONFIRMED
- **Files touched:** `src/renderer/src/lib/canvasView.ts`, `src/renderer/src/canvas/AppChrome.tsx`
- **Roadmap:** none
- **Assigned:** _(left blank for the dev team)_

## Summary
OVERVIEW_FRAME = { padding: 0.3 } is documented as 'intentionally airier than fit'. FIT_FRAME caps maxZoom at 2, but OVERVIEW_FRAME inherits the flow's maxZoom (Z_MAX = 2.5) when rf.fitView(OVERVIEW_FRAME) runs (AppChrome.tsx:256). For a small content cluster, both fits hit their caps: Fit lands at zoom 2.0 while Overview lands at 2.5 — the bird's-eye button zooms IN 25% past the tight-crop button.

## Where
`src/renderer/src/lib/canvasView.ts:45` — module-level `OVERVIEW_FRAME` constant (camera framing presets; consumed by `AppChrome`'s overview button via `rf.fitView(cameraAnim(OVERVIEW_FRAME))`)

## How it triggers
Canvas with a single near-minimum board (e.g. 240x160 after resizing to MIN_BOARD_SIZE) in a typical ~1920px pane: raw overview zoom = 0.4 * 1920/240 = 3.2 → clamped to Z_MAX 2.5; Fit = (1920-128)/240 = 7.5 → clamped to 2.0. Press Fit, then Overview: the camera zooms further IN.

## Verification evidence
Code-path proof: canvasView.ts:37 FIT_FRAME = { padding: '64px', maxZoom: 2 }; canvasView.ts:45 OVERVIEW_FRAME = { padding: 0.3 } (no maxZoom), doc'd at lines 43-44 as 'intentionally airier than fit'. AppChrome.tsx:256 calls rf.fitView(cameraAnim(OVERVIEW_FRAME)); cameraAnim (motion.ts:63-70) adds only duration/ease. Canvas.tsx:825 sets the flow's maxZoom={Z_MAX} (2.5, canvasView.ts:14). Vendored @xyflow/system dist (fitViewport, index.js:431) resolves `options?.maxZoom ?? maxZoom` — i.e. the flow's 2.5 when the preset omits it — then getViewportForBounds clamps zoom = clamp(min(xZoom,yZoom), minZoom, maxZoom) (index.js:746-747). For a near-minimum 240x160 board (MIN_BOARD_SIZE, boardSchema.ts:232) in a ~1920x1080 pane: Fit raw min-zoom = (1080-128)/160 ≈ 5.95 → clamped 2.0; Overview raw = (1080-0.6*1080)/160 ≈ 2.7 → clamped 2.5. Both presets hit their caps and Overview (2.5) > Fit (2.0): the bird's-eye button zooms IN 25% past the tight-crop button, inverting the module's own documented contract. No test contradicts this (canvasView.test.ts only asserts constants).

## Suggested fix direction
Give OVERVIEW_FRAME an explicit maxZoom no greater than FIT_FRAME's (e.g. maxZoom: 1).
