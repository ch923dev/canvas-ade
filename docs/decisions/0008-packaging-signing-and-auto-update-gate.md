# ADR 0008 — Packaging, code-signing, and the auto-update gate (Phase 5)

- **Status:** Accepted (2026-06-15)
- **Context:** Phase 5 (packaging & release) is the release blocker. Most of it can be built and
  verified now, but **code signing requires real-world purchases** (an Apple Developer Program
  membership + Developer ID cert; a Windows Authenticode cert, which since 2023 needs a hardware
  token / cloud HSM) and secrets that only the maintainer can provide. We also must not regress the
  standing security invariant (`electron-builder.yml`, audit `electron-updater-unsigned-latent`):
  **an unsigned app must never pull an update over a feed** — that is a remote-code-execution vector.

## Decision

Ship Phase 5 as **release-ready, signing-staged**: everything except the certs is built now and
structured so that going from unsigned → signed + auto-updating is purely **adding secrets**, with
no code or workflow edits.

1. **App identity.** Real version (`0.1.0`), an on-brand generated icon (`build/icon.png`, the
   outline-diamond mark; regenerated by `scripts/gen-icon.mjs`), and macOS hardened-runtime
   entitlements (`build/entitlements.mac.plist` + `*.inherit.plist`).
2. **Update feed.** `electron-builder.yml` `publish:` is set to the GitHub provider. This both
   fixes a latent bug — `publish: null` silently nullified `--publish always`, so the release job
   went green while uploading **zero** assets — and bakes `app-update.yml` so electron-updater has
   a feed. **Configuring the feed does NOT by itself enable auto-update.**
3. **Auto-update gate (the crux).** electron-updater is wired in `main` (`src/main/autoUpdate.ts`,
   dependency-injected + unit-tested; renderer surfaces status through the existing toast channel),
   but the `checkForUpdates` path is fenced behind a **build-time constant `__ENABLE_AUTO_UPDATE__`**
   (`electron.vite.config.ts` `define`). It is `true` only when the build sets `ENABLE_AUTO_UPDATE=1`.
   In an unsigned build the constant is `false` and esbuild **dead-code-eliminates the entire updater
   path out of the bundle** (verified: no `electron-updater` reference survives; a signed build keeps
   the full wiring + the dynamic `import('electron-updater')`). The security invariant is therefore
   enforced **by the compiler**, not by convention.
4. **CI signing, staged.** `production.yml` passes every signing input from a repo secret
   (`CSC_LINK`/`CSC_KEY_PASSWORD` + `APPLE_*` for macOS; `WIN_CSC_*` for Windows) and gates
   `ENABLE_AUTO_UPDATE` on a repo **variable** `AUTO_UPDATE`. Absent secrets → electron-builder
   degrades gracefully to unsigned; the pipeline keeps working. Staging stays unsigned by design.

## Consequences

- **Now:** unsigned installers build on the full matrix; a published GitHub Release uploads real
  assets + `latest.yml`. The packaged app carries no auto-update code at all.
- **To go signed + auto-updating:** add the signing secrets, then set `AUTO_UPDATE=1` *after*
  verifying signing — see [`../contributing/releasing.md`](../contributing/releasing.md). No code change.
- The gate means there is **no window** in which an unsigned build can auto-update: the code that
  would do so does not exist in unsigned bundles.
- **Open / external:** the certs themselves (maintainer purchase). The end-to-end packaged
  auto-update upgrade ("an in-app update upgrades a prior version") can only be verified once a first
  signed release exists.

## Alternatives considered

- **Lazy `require('electron-updater')` at runtime instead of the build gate** — would keep the boot
  safe but still ship the updater code in unsigned builds. The build-time gate is stronger (the code
  is absent) and is what the `define` already gives us; the dynamic `import` is retained inside the
  gated path so signed builds load it lazily and unsigned builds don't reference it at all.
- **Enable the feed + auto-update immediately (no gate)** — rejected: violates the unsigned-update
  RCE invariant.
- **Bundle electron-updater into the main chunk** — rejected: electron-updater's dynamic provider
  `require`s make it fragile to bundle; externalizing + the dynamic import in the gated path is clean.
