## What shipped

Read-only project memory exposed to agents, 🔒 **PASSIVE context only** (no action):
- `canvas://memory` — the project memory index.
- `canvas://board/{id}/summary` — a board's memory summary.

Backed by the sibling Brain/Memory engine's `<project>/.canvas/memory/` (`MEMORY.md` +
`board-<id>.md`). That engine ships on the parallel `feat/context` track and its **write side is
deferred** (M-memory), so the files are usually ABSENT → every read **gracefully empties** to
`{ present: false, text: '' }`, never an error. **The brain was NOT built here — T1.7 only reads its
`.canvas/memory/` shape** (coordinated via the `context-subsystem` memory: `{MEMORY.md, project.md,
board-<id>.md}`).
