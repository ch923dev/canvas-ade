Same response, three ways to tell value types apart. The constraint: Canvas ADE uses one functional accent (#4f8cff) and reserves --ok/--warn/--err for status only — so a 5-color "rainbow" highlighter (what DevTools does, and what the first mocks wrongly did) is off-brand. The accent is spent on keys (the highest-value thing to find); the question is what — if anything — happens to the values.
Option A — accent keys, neutral valuesRecommended
Keys → accent. All values → neutral white. Type shown by a small grey badge. null dim italic; URLs accent. Zero tension with the one-accent rule.

Option B — + muted type tintsOpt-in toggle
Option A plus a desaturated per-type tint on values (mixed ≥45% toward grey). Best raw readability, but it borrows status-adjacent hues → ship only as an off-by-default toggle, never near status dots.
