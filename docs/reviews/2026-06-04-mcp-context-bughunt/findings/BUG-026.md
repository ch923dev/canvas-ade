# BUG-026: Non-null assertion on localServer in SMOKE=exit path → possible null deref

- **Severity:** Low
- **Category:** correctness
- **Status:** CONFIRMED (adversarially verified)
- **Files touched:** `src/main/index.ts`
- **Roadmap:** none
- **Collides with:** BUG-025
- **Target tree:** `origin/main` (line numbers as of `integration/mcp-on-main`@bc236ee, `src/` identical to origin/main)
- **Assigned:** _(unassigned)_

## Summary
At line 270, `localServer!.url` uses a TypeScript non-null assertion. `localServer` is assigned inside a `try/catch` block at lines 156-165: if `startLocalServer()` throws, `localServer` remains `null`. The `SMOKE !== 'mcp'` branch (which executes this line) does not check whether `localServer` is non-null before accessing `.url`, and will throw an uncaught runtime error that the `process.on('uncaughtException')` handler at line 365 catches — calling `crashShutdown(1)` and producing an exit code 1 smoke failure instead of a clean `SELFTEST_DONE` line.

## Where
`src/main/index.ts:270`

## How it triggers
Running `CANVAS_SMOKE=exit pnpm start` on a machine where the local preview server fails to bind (e.g. firewall blocks loopback on 127.0.0.1, or EACCES). The catch block swallows the bind error, the smoke `else` branch runs, and `localServer!.url` throws `TypeError: Cannot read properties of null`.

## Verification evidence
Code-path argument:

1. `localServer: LocalServer | null = null` (line 43, Z:/canvas-ade-integration/src/main/index.ts)
2. Lines 155-165: `localServer` is assigned inside a `try/catch`. If `startLocalServer()` throws, the `catch` block logs the error and continues — `localServer` remains `null`, and `defaultPreviewUrl` remains `''`.
3. Line 270: `const ok = await runSelfTest(mainWindow!, localServer!.url)` — the non-null assertion `!` bypasses TypeScript's type guard. At runtime, if `localServer` is `null`, this evaluates `(null as LocalServer).url` which throws `TypeError: Cannot read properties of null (reading 'url')`.
4. The `process.on('uncaughtException')` handler (line 365, per the candidate description) then calls `crashShutdown(1)`, emitting an exit-code-1 failure with no `SELFTEST_DONE` line — which makes the smoke suite report a crash instead of a graceful degraded result.

The safe value is already in scope: `defaultPreviewUrl` (line 155) is set to `localServer.url` inside the try block (line 158), and falls back to `''` on failure. Line 196 already uses `defaultPreviewUrl` correctly when calling `registerPreviewHandlers`. Line 270 redundantly re-reads `localServer.url` instead of reusing `defaultPreviewUrl`, introducing the null-dereference.

The trigger condition (local server fails to bind — EACCES, firewall blocking loopback, fd exhaustion) is realistic in restricted enterprise/CI environments.

## Suggested fix direction
Replace `localServer!.url` at line 270 with the already-computed `defaultPreviewUrl` variable, which is `''` when the server failed and correctly set when it succeeded.
