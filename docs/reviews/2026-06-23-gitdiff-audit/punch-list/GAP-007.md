# GAP-007 — No timeout/abort on the git invocation; a hung git pins the task in "reporting"

**Severity:** Low · **Class:** incomplete-impl (robustness) ·
**Rubric:** edge cases & errors; determinism + performance.

## What's missing / wrong
Neither layer bounds how long the diff may take:
- `boardGitDiff` calls `git.checkIsRepo()` then `git.diff(...)` with no timeout
  (`src/main/gitDiff.ts:21-33`).
- The renderer awaits `api.gitDiff?.(terminalId)` inside the `reporting` phase with no timeout
  (`src/renderer/.../useCommandDispatch.ts:196-205`).

If git hangs (network filesystem, a paused/locked repo, an `.git/index.lock` contention, a
credential prompt on a hook), the IPC invoke never resolves and the task is stuck in `reporting`
("collecting result · computing diff…", `CommandRecapView.tsx:36`) indefinitely. The careful
re-throw in `boardGitDiff` (I5) and the renderer's `catch → ''` (A4) both assume the promise
*settles* — a hang defeats both.

## Files
- `src/main/gitDiff.ts` (preferred: bound at the source)
- `src/renderer/.../command/useCommandDispatch.ts` (defensive renderer-side timeout)

## Fix options
1. **MAIN-side timeout/abort:** give `simple-git` an `AbortController` (or spawn with a kill timer);
   on timeout return `''` (best-effort, matches A4) or a truncated marker. Pairs naturally with
   GAP-002's bounded read.
2. **Renderer-side race:** `Promise.race([api.gitDiff(...), timeout(N) → ''])` so the task always
   advances out of `reporting` even if MAIN stalls. Cheapest standalone mitigation.

## Validation
- Integration: stub `registry.gitDiff` to hang → assert the orchestrator/renderer resolves to `''`
  within the timeout and the task reaches `done`/`failed` (not stuck in `reporting`).

## Collisions
- `src/main/gitDiff.ts` — **shared with GAP-001, GAP-002, GAP-006**. Sequence (GAP-002's read
  rewrite is the natural home for the abort).
- `useCommandDispatch.ts` — disjoint from the other gaps (renderer dispatch hook).
