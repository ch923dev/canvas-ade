# GAP-002 ŌĆö The 100 KB clamp does NOT bound MAIN memory; the whole diff is buffered first

**Severity:** Medium ┬Ę **Class:** incomplete-impl / misleading-invariant ┬Ę
**Rubric:** determinism + large-diff performance; security (privileged MAIN process).

## What's missing / wrong
`GITDIFF_MAX_BYTES = 100_000` is applied in `mcpOrchestrator.gitDiff` **after** `boardGitDiff`
returns the full string (`src/main/mcpOrchestrator.ts:941-950`). `boardGitDiff` calls
`simple-git`'s `git.diff(...)`, and **simple-git uses a streaming `child_process.spawn` that
accumulates every stdout chunk into memory with no `maxBuffer` cap** ŌĆö so the entire diff of a
large/hostile working tree is fully materialized in the MAIN process before a single byte is clamped.

The clamp's own docstring claims it guards against "a large/hostile working tree" (`mcpOrchestrator.ts:67-69`)
ŌĆö but it only bounds the **downstream payload** (chip / view-diff / agent), not MAIN's resident
memory. A multi-GB diff (generated/binary churn, a hostile repo) can spike or OOM MAIN regardless of
the 100 KB cap.

## Demonstrated (this audit)
Source inspection of the installed `simple-git@0.13`-era `dist`: the git task plugin is `spawn`-based
(`spawn.after`, `spawned.stdout?.on("data", ŌĆ”)` accumulation) ŌĆö no `maxBuffer`, no byte ceiling. So
`git.diff()` returns the complete output no matter its size; the clamp runs only afterward.

## Files
- `src/main/gitDiff.ts` (where the read happens ŌĆö add the real bound here)
- `src/main/mcpOrchestrator.ts` (clamp + its overstated docstring)

## Fix options
1. **Bound the read at the source:** stream the diff and abort once > N bytes (e.g. spawn `git diff`
   directly with an output cap, or pass `simple-git` an `AbortController` / size-guarded reader), so
   MAIN never holds more than ~the cap. Mark the result truncated.
2. **Pre-size cheaply:** run `git diff --stat`/`--numstat` first; if the total is huge, return a
   truncated/marker result without materializing the full unified diff.
3. At minimum, **correct the docstring** to state the clamp is a payload bound, not a memory guard
   (so the invariant isn't trusted falsely) ŌĆö but this alone leaves the resource gap open.
