# GAP-006 — Diff is HEAD-relative + repo-wide, not scoped to the agent's work or its cwd subtree

**Severity:** Low · **Class:** semantic limitation (honest but coarse) ·
**Rubric:** correctness vs intent (what the result zone claims to show).

## What's missing / wrong
`boardGitDiff` runs `git diff HEAD` from the board's cwd (`src/main/gitDiff.ts:21,26`). Two
consequences for the "what did this worker change?" result zone:

1. **Pre-existing changes are conflated.** If the repo already had uncommitted edits before the agent
   ran, the chip/view-diff attribute them to the worker. The diff is "working tree vs HEAD", not
   "vs the state when the agent started."
2. **cwd subtree is not scoped.** `git diff` is repo-wide regardless of cwd — if a board's cwd is a
   subdirectory, the diff includes changes elsewhere in the repo. (Demonstrated implicitly: git's
   `diff` ignores cwd for pathspec unless `.` is passed.)

The UI label is honest (`git diff HEAD`, `TaskCard.tsx:246`), so this is **Low** — but it can
mislead a user reading the result zone as "this agent's diff".

## Files
- `src/main/gitDiff.ts` (optional pathspec / base-ref)
- `src/renderer/.../TaskCard.tsx`, `CommandRecapView.tsx` (label clarity)

## Fix options
1. **Label only** (smallest): keep `git diff HEAD` but make the result-zone copy say
   "working-tree changes (vs HEAD, whole repo)" so it's not read as agent-attributed.
2. **Snapshot a base** at spawn: record the worker's `HEAD`/stash baseline and diff against it for a
   true "since the agent started" view (larger; touches the orchestrator spawn path).
3. **Scope to cwd subtree:** pass a `.` pathspec so a subdir board reports only its subtree (debatable
   — agents at repo root would be unaffected; document the choice).

## Validation
- If labels only: snapshot/RTL assert the corrected copy.
- If base-snapshot: integration test — pre-existing edit + agent edit → assert only the agent's edit
  appears.
