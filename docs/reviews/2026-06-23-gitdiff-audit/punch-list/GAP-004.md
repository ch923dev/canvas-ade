# GAP-004 — Stale comments deny the `git_diff` MCP tool that the pinned package now ships

**Severity:** Low · **Class:** stale-doc / wiring + coverage ·
**Rubric:** wiring (no dead paths, reachable end-to-end); tests that assert real behavior.

## What's missing / wrong
Multiple in-repo comments assert the agent-facing tool does **not** exist yet:
- `src/main/mcp.ts:104-105` — "the `@expanse-ade/mcp` package registers no `git_diff` tool yet, so
  it is not wire-reachable."
- `e2e/gitDiff.e2e.ts:15` — "The `@expanse-ade/mcp` package registers no `git_diff` MCP TOOL yet
  (see PR-2b)."
- `src/main/mcpRegistry.ts:91` & `mcpOrchestrator`/`mcp.ts` PR-2b mentions — same "deferred" framing.

But the **pinned** dependency (`package.json` → `@expanse-ade/mcp@^0.13.0`, installed **0.13.0**)
**does** register it:
```
node_modules/@expanse-ade/mcp/dist/index.js:19   var TOOL_GIT_DIFF = "git_diff";
…:612  // src/server/tools/gitDiff.ts  → registerGitDiff(server, orchestrator)
…:624  const diff = await orchestrator.gitDiff(args.boardId);
```
The app mounts this server via `createMcpHttpServer` and passes the same orchestrator
(`src/main/mcp.ts:160-174`). So the agent-facing path is in fact registered and routes to
`orchestrator.gitDiff` — the comments are **stale** (they predate the 0.11.0 ship + the ^0.13.0 pin).
The command-board README already records "PR-2b … DONE — shipped 0.11.0 (app-pin bump deferred)"
(`docs/research/2026-06-15-command-board/README.md:143`), and the pin is now ≥ 0.11.0 → the
"app-pin bump deferred" caveat is also satisfied.

Risk: a future maintainer reads "not wire-reachable" and assumes the tool is dead, or fails to gate
it, or duplicates the work. There is also **no app-side test** asserting the package's `git_diff`
tool actually reaches `orchestrator.gitDiff` over the wire (the e2e uses the CANVAS_E2E seam, not the
MCP tool).

## Fix options
1. **Reconcile the comments** to state the tool IS registered (pkg ≥ 0.11.0, pinned ^0.13.0) and
   routes to `orchestrator.gitDiff`; note the remaining gating (per `[[mcp-not-wired-to-terminals]]`,
   terminal agents aren't auto-connected, so reachability depends on a minted token + scope).
2. **Add a test** that drives the `git_diff` MCP tool through the mounted server (or the package's
   tool registration with a stub orchestrator) → asserts it calls `orchestrator.gitDiff(boardId)` and
   returns the text content. Confirm the required token tier/scope.
