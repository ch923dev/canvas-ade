# GAP-005 — Real-git behavioral cases + rendered consumption are untested

**Severity:** Low · **Class:** test-coverage (stale/thin tests) ·
**Rubric:** behavioral coverage of every intended case; e2e actually exercises the feature.

## What's missing / wrong
The diff cases the rubric enumerates are only **partially** tested, and several real-git behaviors
are asserted nowhere:

| Case | Current test | Gap |
|------|--------------|-----|
| no-commits-yet fallback (`git diff`) | mock only (`gitDiff.test.ts:35`) | never run vs real git |
| deleted tracked file | none | git renders `deleted file mode`; unverified end-to-end |
| binary file | none | `Binary files … differ` → chip "+0 −0"; unverified |
| clean repo → `''` | mock only (I1/I2) | no real clean-repo assertion |
| `parseDiffStat` vs **real** git output | hand-written SAMPLE only | parser never fed actual `git diff` text |
| rendered chip / view-diff (TaskCard/Recap/Groups) | none | `parseDiffStat`/`hasDiff` are unit-tested, but no test asserts the `+N −M` chip or the view-diff `<pre>` render |

The e2e covers exactly one happy path (modified + intent-to-add). A regression in the fallback, the
delete path, or binary handling would ship green.

## Files
- `src/main/gitDiff.test.ts` (or a new `gitDiff.integration.test.ts` using a real temp repo)
- `e2e/gitDiff.e2e.ts` (extend the seeded repo to cover delete + binary + clean)
- `src/renderer/src/lib/diffStat.test.ts` (feed real `git diff` fixtures)
- optionally an RTL test for `TaskCard`/`CommandRecapView` rendering the chip

## Fix / build steps
1. Add an integration test that builds a real temp repo (mirror the e2e's hermetic
   `GIT_DIR`/`GIT_WORK_TREE` pinning, `e2e:22-41`) and drives `boardGitDiff` through: no-commits
   fallback, deleted file, binary file, clean (empty) — assert the raw output shape each time.
2. Add `diffStat.test.ts` fixtures captured from real `git diff` (delete, binary, rename) and assert
   the counts (coordinate with GAP-003's oracle).
3. Extend `e2e/gitDiff.e2e.ts` with a deleted + a binary file in `beforeAll`; assert
   `deleted file mode` and `Binary files` appear (coordinate with GAP-001's untracked case).
