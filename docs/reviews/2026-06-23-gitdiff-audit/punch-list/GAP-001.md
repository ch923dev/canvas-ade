# GAP-001 — Untracked (never-`git add`-ed) files are invisible in the diff

**Severity:** Medium · **Class:** incomplete-impl (diverges from user expectation; see intent A1) ·
**Rubric:** behavioral coverage (added/renamed files).

## What's missing / wrong
`boardGitDiff` runs `git diff HEAD` (`src/main/gitDiff.ts:26`). `git diff HEAD` reports **tracked**
changes only. A new file the agent created but never staged is **omitted entirely**, and an unstaged
rename (rm old + create new) shows only the *deletion* half — the new file is untracked → invisible.

For the Command board's "what did this worker change?" result zone, this is a real blind spot: agents
frequently write new files without `git add`. The chip + view-diff then under-report or show nothing.

## Demonstrated (real git, this audit)
```
# clean commit, then unstaged edits:
modify tracked.txt · rm torename.txt + create renamed.txt · create untracked.txt
$ git diff HEAD
  diff --git a/torename.txt …  deleted file mode …  -old      # delete shown
  diff --git a/tracked.txt  …  -b / +B-CHANGED               # modify shown
  # renamed.txt: NOT shown (untracked)
  # untracked.txt: NOT shown (untracked)
```
Contrast: the e2e only passes because it uses `git add -N fresh.txt` (intent-to-add) — a real
untracked file would not appear.

## Fix options (pick per intent; read-only must hold — I11)
1. **Document + label** (smallest): rename the chip label from `git diff HEAD` to make the
   tracked-only scope explicit, and note the limitation. No behavior change.
2. **Include untracked, read-only:** union `git diff HEAD` with `git ls-files --others
   --exclude-standard` (list) and/or `git diff --no-index /dev/null <file>` per untracked file to
   synthesize an addition diff. Avoid `git add -N` (mutates the index → violates I11).
3. Defer to the richer **"Changes" Terminal-board panel** (`docs/feature-proposals.md:390`) which is
   already scoped to handle full file lists — see `skipped-roadmap.md`.

## Validation

- e2e: add an untracked file alongside the existing modified+`-N` files; assert presence/absence
