## Browser preview + native WebContentsView

### Features

- **Native WebContentsView preview lifecycle (PreviewManager)** _(user-facing)_ — Keyed Map<id,Entry> of native WebContentsViews owned in MAIN. open/setBoundsBatch/capture/detach/attach/close/closeAll/navigate/goBack/goForward/reload IPC handlers; each view gets contextIsolation+sandbox+nodeIntegration:false and a per-board partition `preview-<id>` to isolate Chromium per-host zoom. webContents.close() (no destroy) on dispose to avoid renderer leak.
  - `src/main/preview.ts:75 ensure(); src/main/preview.ts:208 registerPreviewHandlers; src/main/index.ts:101 registration`
- **Camera-synced native bounds (rAF pump, coalesced batch)** — BrowserPreviewLayer mounted once inside <ReactFlow>; reads live camera via useReactFlow/useOnViewportChange and emits ONE coalesced setPreviewBoundsBatch per frame for all attached boards, diff-skipped via rectsEqual + lastZoom. Self-stopping after 4 idle frames. Native rect = device-stage rect mapped world→screen via worldRectToScreen + measured paneOffset.
  - `src/renderer/src/canvas/boards/BrowserPreviewLayer.tsx:195 flushBatch; :212 startPump; :267 useOnViewportChange; src/main/preview.ts:224 setBoundsBatch`
- **Detach + snapshot for motion / LOD / over-cap** _(user-facing)_ — On move start, capture each live view to a data URL, store as snapshot in previewStore, detach the native view so HTML carries motion (native layer can't be clipped/can't trail). On move end, reattach boards that are live-eligible (zoom>=LOD, on-pane, sized) under MAX_LIVE=4; over-cap eligible boards are closed (renderer freed), below-LOD boards stay snapshot.
  - `src/renderer/src/canvas/boards/BrowserPreviewLayer.tsx:225 beginMotion; :253 endMotion; :143 demoteToSnapshot; src/main/preview.ts:236 capture / :243 detach / :249 attach`
- **Live eligibility + live-cap policy (pure)** — isLiveEligible: zoom>=LOD AND stage w/h>1 AND screenY>=paneTop (native can't paint above pane). pickLive caps the live set first-come at MAX_LIVE. Drives which boards host a live view vs a snapshot.
  - `src/renderer/src/lib/previewPlan.ts:18 isLiveEligible; :35 pickLive; BrowserPreviewLayer.tsx:116 liveEligible`
- **Responsive viewport presets + reflow trick** _(user-facing)_ — Mobile 390x844 / Tablet 834x1112 / Desktop 1280x800 presets. Page held at fixed CSS width W via setZoomFactor = (nodeWorldW/presetW)*camZoom (fitZoomFactor), clamped [0.25,5], so media queries fire at the breakpoint while scaling as a unit with the camera. Viewport selection persists on the board (board.viewport).
  - `src/renderer/src/lib/browserLayout.ts:52 VIEWPORT_PRESETS; src/renderer/src/lib/cameraBounds.ts:83 fitZoomFactor; BrowserBoard.tsx:142 setViewport; BrowserPreviewLayer.tsx:106 zoomFor`
- **Device-frame geometry (HTML chrome around unrounded native rect)** _(user-facing)_ — Pure world-space geometry: contentWellRect (below titlebar+URL bar), deviceFitScale (fit preset aspect, cap MAX_FIT_SCALE 1.1), deviceFrameRect (centered outer rounded frame), deviceStageRect (1px-border inset = native rect), toWorldRect. HTML draws rounded frame/notch/bezel matching the native rect exactly at every zoom.
  - `src/renderer/src/lib/browserLayout.ts:59-129; BrowserBoard.tsx:151 deviceFrameRect; :219 .bb-stage`
- **Editable URL bar + navigation controls** _(user-facing)_ — Per-board URL input with local draft committed on Enter/blur (Escape reverts), one undo checkpoint per commit via beginChange+updateBoard; back/forward (gated by canGoBack/canGoForward) + reload buttons drive preview:goBack/goForward/reload; W×H dims label; connection dot.
  - `src/renderer/src/canvas/boards/BrowserBoard.tsx:124 draft state; :131 commitUrl; :171 url bar; :239 NavBtn`
- **Load lifecycle state surfacing (connecting/connected/load-failed)** _(user-facing)_ — Main pushes did-finish-load/did-navigate/did-fail-load on preview:event; renderer maps to previewStore status, liveUrl, canGoBack/canGoForward, error. Status pill + URL-field conn-dot + device-frame 'Couldn't load' / 'Connecting…' fallback. Error-page double-fire guarded by a `failed` latch in main + a renderer belt-and-suspenders (Bug #5 fix).
  - `src/main/preview.ts:67 emit; :110/:121/:144 wc events; BrowserPreviewLayer.tsx:353 onPreviewEvent; BrowserBoard.tsx:264 DeviceContent; :294 statusPill`
- **Store-driven reconcile of native views with canvas boards** — Imperative useCanvasStore.subscribe (no re-render) maps browser boards→geom; opens new eligible boards, closes removed boards (+clearRuntime), navigates on board.url change, re-pushes bounds+zoom on viewport/geometry change for attached boards.
  - `src/renderer/src/canvas/boards/BrowserPreviewLayer.tsx:272 reconcile; :312 subscribe effect`
- **External-link denial → OS browser** _(user-facing)_ — Each view's setWindowOpenHandler denies in-view navigation and routes target=_blank/window.open to shell.openExternal, so the preview never becomes a general browser/nav target.
  - `src/main/preview.ts:96 setWindowOpenHandler`
- **Loopback localhost preview server (dev+packaged default URL)** — Tiny 127.0.0.1 HTTP server serving a self-reporting responsive page (3 breakpoints, prints innerWidth) used as the default preview URL so the WebContentsView smoke works without a Vite dev server.
  - `src/main/localServer.ts:57 startLocalServer; src/main/index.ts:99 start; :101 used as defaultUrl`
- **Ephemeral preview runtime store** — Zustand store of per-board transient runtime (snapshot, status, liveUrl, canGoBack/Forward, live flag, error), deliberately NOT persisted to canvas.json. selectRuntime/selectLiveCount selectors (live count feeds DiagOverlay).
  - `src/renderer/src/store/previewStore.ts:51 usePreviewStore; :67 selectRuntime; :72 selectLiveCount`
- **E2E debug capture accessors** — Read-only debugCaptureView (attached+empty) and debugViewIds over the live views Map for the in-process e2e smoke harness, closing the #1 live-verify gap that mainWindow.capturePage can't see.
  - `src/main/preview.ts:315 debugCaptureView; :323 debugViewIds`

## Terminal + node-pty bridge

### Features

- **Shell enumeration (OS-aware discovery)** _(user-facing)_ — enumerateShells() probes the filesystem/PATH for available shells, best-default-first (Win: pwsh>powershell>cmd>git bash>wsl>bash>zsh; *nix: $SHELL>zsh>bash>/bin/bash), dedupes by lowercased path, marks head as default, and guarantees a non-empty list via defaultShell() fallback. Exposed to renderer as window.api.listShells() and consumed by the config picker.
  - `src/main/pty.ts:110 enumerateShells; src/main/pty.ts:154 ipcMain.handle('pty:shells'); src/preload/index.ts:50 listShells; src/renderer/src/canvas/boards/TerminalConfig.tsx:35`
- **PTY spawn (control plane)** _(user-facing)_ — pty:spawn resolves the shell (explicit board.shell or default), special-cases Git Bash to add -l -i, spawns node-pty (xterm-256color, cols/rows, cwd=home fallback, inherited env), wires the MessagePort data plane, transfers port2 to renderer via webContents.postMessage('pty:port'), announces running state, and writes launchCommand as the first PTY line. Returns SpawnTerminalResult (pid/state, or spawn-failed+error on throw).
  - `src/main/pty.ts:156 ipcMain.handle('pty:spawn'); src/renderer/src/canvas/boards/TerminalBoard.tsx:161 spawnTerminal; src/preload/index.ts:46`
- **MessagePort data plane (PTY I/O)** _(user-facing)_ — Bidirectional MessageChannelMain port: main posts {t:'data'} (PTY output), {t:'state'} (lifecycle), {t:'exit'} (code); renderer posts {t:'input'} (keystrokes) and {t:'resize'} (cols/rows). Preload re-posts the transferred port into the main world as {__ptyPort,id} since MessagePorts can't cross contextBridge. Renderer matches on board.id and pipes to xterm.
  - `src/main/pty.ts:184 onData/199 onmessage; src/preload/index.ts:99 pty:port forwarding; src/renderer/src/canvas/boards/TerminalBoard.tsx:142 onWinMsg/147 port.onmessage`
- **Lifecycle state machine + identity pill** _(user-facing)_ — PtyState/TerminalState (idle/spawning/running/awaiting-input/exited/spawn-failed) drives the BoardFrame status pill: dot colour per DESIGN §7.1 (--ok running/spawning, --warn awaiting, --err failed, neutral exited/idle) plus a label combining the agent identity. agentIdentity() derives the name from launchCommand head (path+ext stripped) or shell basename.
  - `src/renderer/src/canvas/boards/terminalState.ts:48 statusFor/73 agentIdentity; src/renderer/src/canvas/boards/TerminalBoard.tsx:70,251 wiring`
- **Live xterm terminal rendering** _(user-facing)_ — Mounts @xterm/xterm with FitAddon + WebglAddon, design-token THEME, resolved --mono font; ResizeObserver re-fits on resize; WebGL context-loss handler disposes the addon so xterm falls back to DOM renderer (prevents permanent blank under Chromium's GL-context cap). Always-mounted across LOD so the PTY session survives zoom-out.
  - `src/renderer/src/canvas/boards/TerminalBoard.tsx:88 new Terminal/104 webgl loss/181 ResizeObserver/281 LOD hide`
- **Keystroke focus + key-event isolation** _(user-facing)_ — Body mousedown stops propagation to React Flow and forces xterm.focus() (fixes the 'can't type' bug where RF focuses the node wrapper). A bubble-phase keydown listener stops keys reaching the canvas global handlers (1/0/Esc/Backspace-Delete board-delete) AFTER xterm's textarea consumes Backspace/Enter/arrows/Ctrl-C. nodrag/nowheel classes stop drag/zoom.
  - `src/renderer/src/canvas/boards/TerminalBoard.tsx:139 stopKeys/289 nodrag nowheel/291 onMouseDown`
- **Restart** _(user-facing)_ — Title-bar action: kills the current session (killTerminal → killTree in main), closes the old port, resets the xterm screen, sets spawning, and respawns a fresh shell in place with the same config.
  - `src/renderer/src/canvas/boards/TerminalBoard.tsx:222 restart/260 IconBtn restart`
- **Configure popover (shell/launchCommand/cwd/label)** _(user-facing)_ — Popover edits durable board.shell/launchCommand/cwd/title; Apply calls store.beginChange()+updateBoard, which changes the spawn effect deps and triggers a respawn with new config. Label-only edits change board.title without respawning. Loads the shell list and seeds the default when no shell set. Stops pointer/key propagation; Esc closes.
  - `src/renderer/src/canvas/boards/TerminalConfig.tsx:18 TerminalConfig/45 apply; src/renderer/src/canvas/boards/TerminalBoard.tsx:256 settings IconBtn`
- **Kill-the-tree teardown** — cleanup() is identity-aware (isStaleExit guards a late OLD-process onExit from reaping a NEW session under the same id); killTree() uses taskkill /T /F on Windows and negative-pgid SIGKILL on *nix to reap child process trees (agentic CLIs spawn children). disposeAllPtys() reaps all on app quit. Effect cleanup calls killTerminal on unmount.
  - `src/main/pty.ts:242 cleanup/259 killTree/278 disposeAllPtys/231 isStaleExit; src/renderer/src/canvas/boards/TerminalBoard.tsx:190 effect cleanup/196 killTerminal`
- **E2E test hook registration** — Under the ?e2e=1-gated isE2E() flag, registers the live xterm instance in e2eTerminals keyed by board.id so the self-smoke harness can read the framebuffer (PTY→xterm echo assertion); deregisters on teardown.
  - `src/renderer/src/canvas/boards/TerminalBoard.tsx:122,203; src/renderer/src/smoke/e2eRegistry.ts`

## Planning whiteboard + elements

### Features

- **Whiteboard tool cluster (select/note/check/arrow/pen)** _(user-facing)_ — A mini in-board tool palette rendered in the BoardFrame action slot ONLY while the board is selected. Switching tool changes the well cursor and gesture behavior; clicking a tool clears the selected element. Tools auto-revert to 'select' after a one-shot create (note/check/arrow/pen).
  - `PlanningBoard.tsx:62 (TOOLS), :86 (tool state), :296-325 (actions cluster), :318-321 (setTool)`
- **Sticky note create + inline edit + drag + delete** _(user-facing)_ — 'note' tool drops a tinted, slightly-rotated sticky note centred on the click; body text is editable via an auto-growing transparent textarea; the whole card is a drag handle in select mode; delete via hover X button or Backspace on empty text.
  - `PlanningBoard.tsx:203-209 (create), NoteCard.tsx:44-141, elements.ts:27-44 (makeNote), NoteCard.tsx:56-61 (autosize), :120-123 (backspace-delete)`
- **Free-text element create + inline edit + drag** _(user-facing)_ — Double-click empty well (select mode) drops a chrome-less editable text element anchored at the point; auto-grows width+height; dragged via a slim 6px left gutter so the text stays selectable; delete via X or Backspace on empty.
  - `PlanningBoard.tsx:259-267 (onWellDoubleClick → makeText), FreeText.tsx:38-137, elements.ts:47-49 (makeText), FreeText.tsx:48-55 (autosize)`
- **Checklist element (live progress)** _(user-facing)_ — 'check' tool drops a raised card with editable title, done/total mono count, a 3px accent progress bar that animates with the live done ratio, togglable 16px checkbox items with strikethrough-when-done, add-item button, Enter-to-append, Backspace-on-empty-to-remove (keeps at least one item).
  - `PlanningBoard.tsx:210-215 (create), ChecklistCard.tsx:73-305, elements.ts:52-67 (makeChecklist), :106-153 (toggle/add/remove/setItemLabel), :155-164 (checklistProgress), ChecklistCard.tsx:246-255 (Enter/Backspace keys)`
- **Arrow tool (drag-to-draw bezier)** _(user-facing)_ — 'arrow' tool press-drags a cubic-bezier arrow with a gentle bow + arrowhead marker from start→end in board-local space; renders as a live draft until pointer-up; degenerate (≤4px) arrows are discarded. Committed arrows are selectable (accent highlight + thicker stroke).
  - `PlanningBoard.tsx:216-223 (start), :247-248 (drag update), :279-286 (commit/discard), svgPaths.ts:29-37 (arrowPath), WhiteboardSvg.tsx:72-95`
- **Freehand pen (perfect-freehand)** _(user-facing)_ — 'pen' tool captures pointer points into a flat board-local list, renders a live draft outline, and commits a filled-outline stroke (vendored perfect-freehand, simulatePressure off) when ≥4 coords (≥2 points). Committed strokes are selectable.
  - `PlanningBoard.tsx:224-231 (start), :249-252 (accumulate), :287-292 (commit ≥4), svgPaths.ts:20-26 (STROKE_OPTIONS), :45-54 (strokeToPath), pen.ts:58-72`
- **Screen→board coordinate mapping (÷zoom)** — Maps screen pointer positions to board-local space (subtract well origin, divide by camera zoom) so notes/strokes land under the cursor at any zoom; falls back to zoom 1 on non-finite/non-positive zoom. Load-bearing — the documented Planning gate.
  - `pen.ts:46-52 (screenToBoard), PlanningBoard.tsx:112-122 (toBoard), :84 (live zoom from RF store)`
- **Element drag with single-undo-checkpoint** — Moving a note/text/checklist captures pointer on the well, renders a transient live position via dragPos state, and commits the final position to the store ONCE on pointer-up (beginChange snapshots at drag start) so a whole drag is one undo step rather than one mutation per frame.
  - `PlanningBoard.tsx:97 (dragPos), :178-190 (startElementDrag + beginChange + well capture), :243-246 (transient), :273-278 (commit once), :333 (viewElements)`
- **Element selection + keyboard delete (vector elements)** _(user-facing)_ — Clicking a committed arrow/stroke selects it (accent highlight) and focuses the well; Delete/Backspace on the focused well removes the selected element with an undo checkpoint.
  - `WhiteboardSvg.tsx:81-84,104-107 (onSelect), PlanningBoard.tsx:87 (selectedElId), :359-367 (well keydown delete), :390-393 (onSelect→focus)`
- **Per-board arrowhead marker namespacing** — Each Planning board gets a unique SVG <marker> id (pl-arrowhead-<boardId>) plus a selected variant so multiple Planning boards on one canvas never collide on a shared DOM id.
  - `svgPaths.ts:40-42 (arrowheadMarkerId), WhiteboardSvg.tsx:37,57-69 (defs)`
- **Note tint palette + deterministic rotation** _(user-facing)_ — Four low-chroma tints (yellow/blue/green/plain) cycled by note index, each with a fill+edge, plus a small alternating ±~1° rotation per index for a hand-placed feel.
  - `tints.ts:20-25 (NOTE_TINTS), :31-35 (noteRotation), elements.ts:40-41`
- **Element persistence via store + schema** — All elements are stored board-local on board.elements and persisted through store.updateBoard; the schema validates each element kind on load (round-trip), enabling future canvas.json I/O.
  - `PlanningBoard.tsx:107-110 (commit→updateBoard), boardSchema.ts:251-298 (assertPlanningElement), canvasStore.ts:97-119 (updateBoard with PATCHABLE_KEYS allow-list)`
- **Draw-through-card gesture fallthrough (#6)** _(user-facing)_ — In a draw mode (pen/arrow), card pointer-down does NOT stop propagation, so a stroke/arrow can begin on top of a note/text/checklist and reach the well; in select mode cards swallow the press for editing/drag.
  - `NoteCard.tsx:78-86, FreeText.tsx:68-70, ChecklistCard.tsx:116-118,160-162,216-218,243-245, PlanningBoard.tsx:196-199`
- **13px dot-grid sketch surface + empty-state hint** _(user-facing)_ — The content well paints a finer 13px radial-dot grid to read as a sketch surface and shows an empty-plan hint ('pick a tool…' when selected, 'empty plan' otherwise) when there are no elements.
  - `PlanningBoard.tsx:375-376 (grid), :444-457 (empty hint)`
