- **🐞 Browser-board bug found by the harness (TRACK in Track 2):** a dead/refused URL ends showing **`connected`** because Chromium's error page fires `did-finish-load` AFTER `did-fail-load`, so `BrowserPreviewLayer` overwrites `load-failed` with `connected` (and the error page is non-blank → capture isn't empty). Fix: in `preview.ts`/`BrowserPreviewLayer`, don't let an error-page `did-finish-load` override a prior `did-fail-load` (e.g. track a failed flag or check the HTTP response code).
