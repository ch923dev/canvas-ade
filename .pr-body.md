## What

**New `canvas/Menu.tsx` + `canvas/menuPlacement.ts` (pure clamp) + `Menu.test.tsx` (23 tests).** The shell owns: body portal · measure-then-clamp before paint · Escape / outside-pointerdown / resize dismissal · `menuitem` roving tabindex + ArrowUp/Down/Home/End (wrap, skip disabled) + Tab-closes · focus restore on close · the **ADR 0002 detach-live-previews-while-open token** (PREV-C ref-count, `menu.e2e.ts` still hard-asserts detach→reattach).

**Migrated (all six):** board ⋯ menu (BoardFrame) · Tidy picker (AppChrome) · project switcher (AppChrome) · GroupContextMenu · ElementContextMenu · terminal well context menu (an ElementContextMenu instance — rode along).

`clampMenuToViewport` unifies ElementContextMenu's flip-at-the-pointer, BoardMenu's trigger-anchored flip-above, and D0-4's maxHeight scroll cap; point + trigger anchor modes.

## Net-new behavior

- `role="menuitem"` on board-⋯ items + project-switcher rows (the #107 gap); arrow-key nav + roving tabindex in **every** menu (A8)
- Viewport clamp everywhere — switcher gains the bottom-edge flip, Tidy gains the maxHeight cap
- **ElementContextMenu now detaches live previews while open** — closes its documented native-occlusion limitation (planning + terminal right-click menus)
- Project switcher detaches previews while open (was uncovered)

- Outside-close = capture-phase listener + `contains()` + trigger exclusion → a `stopPropagation` anywhere can't strand an open menu

## The bug the matrix caught (worth a read)

First Windows run failed `groups.e2e.ts:150` — Escape stopped closing the group menu. Root cause: the shell's dismissal effect depended on `[onClose, anchor]`; callers pass inline closures, so any owner re-render re-subscribed the window listener. The kicker: `useCanvasKeybindings`' earlier window listener maps Escape→`clearSelection`→zustand write, and `useSyncExternalStore` flushes that re-render **synchronously mid-dispatch** — a listener removed mid-dispatch never fires (DOM spec), so the menu's own Escape handler was silently skipped. The old per-menu copies dodged it only because they listened on `document` (bubbles before `window`). Fix: dismissal listeners are **mount-stable**, reading `onClose`/`anchor` through refs; unit regression added (no re-subscribe on prop identity churn).
